## The Multi-Agent Workflow: How We Actually Build

Picture this: **Argos**, our chief of staff agent, ingests your requirements. It spins up specialists—**Codie** for code, **Talos** for strategy—each with domain souls (SOUL.md files defining personality and expertise). They communicate via structured briefings in shared inboxes, iterating until Definition of Done: PR merged, CI green, deployed.

No hallucinations. Every task has acceptance criteria, verification commands, and memory layers for continuity. It's like GitOps but for cognition.

### The Three Layers

**Layer 1: Orchestration**  
Argos scopes tasks, assigns to specialists, unblocks issues. Handles budget allocation, priority conflicts, Definition of Done validation.

**Layer 2: Execution**  
Agents spawn sub-agents for parallelism. Codie writes code, tests first (TDD). Claude Code handles complex architecture. Codex handles volume work.

**Layer 3: Verification**  
Tests first, code second, human review last. No claim of "done" without verification commands proving it.
